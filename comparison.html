Differences with coreboot

While Librecore shares much of its source code with coreboot, and also collaborates with coreboot, and even sends patches to coreboot, there are also some differences between the two projects.

While the differences are subject to the evolution of both projects and their members, so far we identified some below.

Freedom

To support some devices, coreboot uses Huge silicon manufacturer provided blobs like AMD's PI and Intel's FSP.
Such blobs are responsible for enabling most of the hardware, which is what coreboot is supposed to do in the first place. This relegates coreboot to a wrapper of such blobs, leaving it very few things left to do.

Because of that Librecore chose not to use such blobs. As a result of that, the hardware requiring such blobs is supported by coreboot but not librecore.

When such blobs are optional, not having the option to use them is less confusing for users and/or developers.

Maintanability

Coreboot uses Silicon manufacturer provided free software source code (Like AMD's AGESA) that don't follow the same design or code practices than the community is used to:
Such source code often contains a layer to abstract the OS or, here the boot firmware project it's used with. This is a well known issue[1] in free software projects that interact with hardware. While such code is really valuable as it provides a good working reference, but it has huge maintainability issues.
The Linux kenrel also has to deal with this issue, and because of the maintability issues, decided not to use that kind of code as-is.
